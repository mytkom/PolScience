What do the scores mean?
  • Keywords — lexical match between your query and each scientist’s indexed text (publication titles/keywords or profile fields, depending on mode).
  • Semantic — embedding similarity: how close the scientist’s text is to the query in meaning, not just shared words.
  • Community — co-author network score used in Final: by default, personalized PageRank seeded from top keyword matches. If query PPR is disabled and precomputed GEXF graphs are loaded, the Community weight uses static co-auth PageRank instead.
  • Co-auth degree, Network rank, Cluster — background network context, not query scores. Co-auth degree counts how many distinct co-authors the profile has in the search index’s co-authorship graph (any shared publication). Network rank is global PageRank on that same graph: how well connected the profile is in the overall network, regardless of your keywords. Cluster names the modularity community the profile was assigned to in offline graph exports (when GEXF is loaded). These columns do not affect Final while query PPR is enabled; only the Community score enters fusion.

Keywords, Semantic, and Community are raw scores used in fusion (BM25, cosine similarity, PPR or static PageRank). Profiles with no indexed text are excluded. Zero keyword score is allowed in the pool. Final uses min–max normalized signals over that pool and your fusion weights.
