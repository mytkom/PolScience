# Polish Science Database
We aim in this project to create a database of polish science (scientists with their publications, project, patents and employments). We want not only to use central database like Ludzie Nauki (People of Science), but also enrich it with repository pages of specific universities and email information from universities' contact books (if available online).

Second goal is to create a search tool that will be useful for Research4Revenue search for scientists for new cohort. They must match specific classical criteria (number of publications, projects in last 2 years etc.) and semantic search would be useful for filtering a desired niche of scientists, by specific reasearch problem, which they are tackling.

Finally, we've created a set of graphs on the basis of our database and share their binary representation for further studies.

## Design decisions
We decided that for our volume of the data (few GBs) SQLite database is enough and has the most flexibility. There is no need to host it anywhere or worry about replicas etc. Relational database was a natural choice as there are many relations between entities (e.g. scientists many-to-many with publications, institutions, projects, patents).

For search tool we decided to use FastAPI with a simple GUI available in browser. It can be run locally, but can be in the future hosted somewhere and be available remotely. Once again, greater simplicity and flexibility.

For graph generation we use Python with NetworkX for the full pipeline from graph construction to metric computation and GEXF export. Gephi is used for interactive exploration of the generated graphs, as it is a well-established tool for visual analysis of large and complex networks.

## Project structure and components
For a detailed documentation follow the subdirectory of interest. Here is just a quick overview:
- **Ludzie Nauki Scraper** (./LudzieNaukiScraper) - a 4-pass Python scraping script. It uses Ludzie Nauki JSON API and saves results into a SQLite database. Rate-limiting, concurrent workers and random wait between requests features are available.
- **PolUni RAD-on ETL** (./PolUni) - A Java terminal application that extracts Polish university and institution data from RAD-on / POL-on APIs, applies user-selected filters, and exports institution metadata and student count statistics into CSV files, which are then used to enrich organisations' database entries.
- **Email enrichment PoC** (./PWrEmailDataEnrichment) - an autogenerated scraper for PWr contact book and Python script merging it into the DB. We add it as a proof of concept methodology for further extension of our DB.
- **Graph Generation** (./graphGeneration) - a Python module that builds three graph representations of the Polish academic network (researcher co-authorship, institution collaboration, specialty co-occurrence) from the SQLite database, computes structural metrics (PageRank, betweenness centrality, community detection, etc.), and exports them to GEXF format for exploration in Gephi. Generated graph visualisations are available in ./graphGeneration/graphs/.
- **Expert search app** (./ExpertSearchApp) - BM25 + semantic + PPR ranking over Ludzie Nauki profiles; FastAPI UI and CSV export (see subfolder README).
