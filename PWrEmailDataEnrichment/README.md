# WebScraper-based PWr email scraping and enrichment
We've used another's group tool, the WebScraper, which is a LLM-based auto scraping tool. It automaticaly produces a script for scraping, which saves data into a CSV file. Utilising this we were able to scrape the whole Politechnika Wrocławska's contact book and add email to our database if first name and surname are matched to person in our database. It is a proof of concept that our database can be easily expanded with such a tool.

Scraping scripts is `people.js`, the output data is in `people.json`. We also add a script `update_pwr_emails.py`, populating emails into given SQLite database.

# Web Scraper — Generated Script

This folder contains a standalone web-scraping script generated by **WebScraper**.
It drives a real Chrome browser with [Puppeteer](https://pptr.dev/) and prints the data it collects.

## Requirements

- **Node.js 18+** — check with `node -v`
- **Google Chrome** (or let the install step below download a bundled Chromium)

## 1. Install dependencies

From this folder, run:

```bash
npm init -y
npm install puppeteer-extra puppeteer-extra-plugin-stealth puppeteer
```

> `puppeteer` downloads its own Chromium. To use a Chrome you already have, skip it and set `CHROME_PATH` (see below).

## 2. Run it

```bash
node workflow.js
```

The script starts at `https://pwr.edu.pl/pracownicy/strefa-pracownika/wyszukiwarka-pracownikow`, runs each step, then prints the results.

## Output

Collected data is printed to the terminal when the run finishes.

To save it to a file, either redirect the console output:

```bash
node workflow.js > results.txt
```

…or write JSON directly — see *Save results to a file* below.

## Where to make changes

All edits are in `workflow.js`:

- **Start URL / target page** — the first `await page.goto("…")` inside `run()`.
- **What gets extracted (CSS selectors)** — look for `{ value: "<selector>", type: "css" }` in the extraction steps and edit the selector strings.
- **Workflow variables** — if you add any, they appear as `let` declarations at the top of `run()`.
- **How many pages to scrape** — the `for (let i = 0; i < _rep_total; i++)` loop; change the repeat count (or the page math) to visit more/fewer pages.
- **Field cleaning / splitting** — fields are post-processed in the `__ftMaterializeRow(...)` call; tweak the `transforms` / `split` specs there.
- **Show the browser window** — set `headless: true` to `false` in `puppeteer.launch({ … })`.
- **Use your own Chrome** — set the `CHROME_PATH` env var, or edit `executablePath` in `puppeteer.launch`.
- **Window size** — `page.setViewport({ width, height })`.
- **Timeouts** — the `timeout:` values on navigation / waits (milliseconds).
- **Save results to a file** — at the end of `run()` add:

```js
require('fs').writeFileSync('results.json', JSON.stringify(__results__, null, 2));
```

## Troubleshooting

- **"Could not find Chrome" / launch fails** — install Chrome and set `CHROME_PATH`, or run `npx puppeteer browsers install chrome`.
- **Empty results** — the site's markup probably changed; update the CSS selectors.
- **Timeouts / slow pages** — increase the `timeout:` values, or add a `Wait` after navigation.
- **Blocked / bot detection** — try running non-headless, slow the script down, and respect the site's `robots.txt` and terms of service.
